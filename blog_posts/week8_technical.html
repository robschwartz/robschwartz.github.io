Running on Rails

July 3rd 2014

When I heard about the Ruby language, I first heard of it as "Ruby on Rails". Little did I know Ruby and Rails were 2 separate entities! Ruby is the back-end programming language and Rails is an open source web application framework which runs on and is built on Ruby.

Great! So, what does that mean?

Rails is the application that helps Ruby become live on the web. It makes it easier for the program to run and helps programmers by offering templates to get them started with a web page. Rails is a Model-View-Controller (MVC). MVC is a design that keeps different parts of the application separated into, you guessed it, Model/ View/ and Controller. The Model interacts with the Database, the View puts the HTML code to work, and the Controller handles the interaction with the web server.

Understood, now how do I use it?

Following the tradition of being user friendly, Rails offers the complete package just by downloading the Rails gem on to your computer. For complete step by step instructions and a complete package of Rails, you can visit installrails.com. For simpler installation just run this code in the terminal:

$ gem install rails

(full instructions here)

I hope this guide helps you understand where Ruby and Rails differ and how/ why they are used together!
